CursorRuler

Generate and maintain .cursorrules files for LLM-assisted development

Overview

CursorRuler is a CLI tool that helps you generate and maintain .cursorrules files for your projects. These files contain your project structure and code in a format that's optimized for Large Language Models (LLMs), enhancing AI-assisted development.

Requirements

  • Node.js ≥ 18.0.0
  • pnpm ≥ 9.0.0

Installation

pnpm add -D cursorruler

Quick Start

npx cursorruler -i "*.js" -i "*.json" -e "node_modules/**" -w

Command Line Options

  • -i, --include <patterns...>: File patterns to include
  • -e, --exclude <patterns...>: File patterns to exclude
  • -w, --watch: Watch for file changes
  • -r, --refresh: Auto-refresh every 5 seconds
  • --no-gitignore: Ignore .gitignore patterns

Configuration

Create a .cursorruler.json file in your project root:

{
    "include": ["*.js", "*.json"],
    "exclude": ["node_modules/**"],
    "watch": {
        "ignored": ["node_modules", ".git"],
        "persistent": true
    }
}

Features

  • Automatic file watching and updates
  • Smart file minification with fallback strategies
  • Gitignore integration
  • Support for multiple file types:
    • JavaScript (.js)
    • TypeScript (.ts, .tsx)
    • JSON (.json)
    • YAML (.yml, .yaml)
    • TOML (.toml)
    • HTML (.html)
    • CSS (.css)
    • XML (.xml)

Examples

Basic Usage

# Include all JavaScript files
npx cursorruler -i "*.js"

# Include JavaScript and exclude test files
npx cursorruler -i "*.js" -e "**/*.test.js"

# Watch mode with multiple patterns
npx cursorruler -i "*.js" -i "*.json" -e "*.test.js" -w

Best Practices

  • Always exclude sensitive files and directories
  • Use specific include patterns to minimize unnecessary file processing
  • Consider using watch mode during development
  • Regularly update your .cursorrules file to maintain sync with your codebase
Note: Make sure to add .cursorrules to your .gitignore if you don't want to commit the generated file.
